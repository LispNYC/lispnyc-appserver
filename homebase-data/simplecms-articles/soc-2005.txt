Proposal Ideas:

* Optimizing the SBCL calling convention on the x86(-64) architectures
* Peephole optimizations for SBCL
* Loop optimizations for SBCL
* Protocol extensions for CLX
* Spatial indexing for graphical user interfaces
* Low level Common Lisp interface to Win32 API for GCL 
* Multithreaded GCL on Windows and Unix
* Lilac: UFFI interface auto-generation
* Lisp Resource Kit multi-media indexing and display system
* Complete CLISP's native threading
* Extend CMUCL for systems programming
* Addition of a Lisp language to WikiTalk
* Modify storage requirements of Persistent Lisp Objects
* Programmable Protein Structure Environment
* Clojure Lisp Compiler: Compiles Common Lisp to Java and C# (formally DotLisp)
* Failover Feature for the BKNR Transactional Engine
* Distributed Versioning based on Epistemic Monotonic Logic
* Dynamic Persistent Web Proxy
* Session-aware Web Server Multiplexer
* Upgrade CL-glade
* Hyperreal Dictionary of Mathematics
* Wiki-like collaborative hypertext system in CL
* Embedding a Common Lisp interpreter in the Apache web server
* Parsing Expression Grammars in PLT Scheme
